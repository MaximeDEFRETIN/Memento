La ligne de commande

Gestion de documents

Créer un dossier :


    mkdir //permet de créerun dossier

    mkdir -p dossier/sous-dossier //créer un dossier avec un sous-dossier

Voir le conteu d'un dossier :


    ls //permet de voir les dossiers

    ls dossier/sous-dossier (ex:home/maxime) //permet de voir le contenu de ces dossiers

    ls -l //affiche les dossier en version longue

    ls -a //affiche les dossiers cachés

    ls -al //afiches les dossiers en version longue, y compris ceux cachés

Afficher le chemin d'un dossier :


    pwd //ffiche le chemin du dossier dans lequel on se trouve

Passer en mode root :



    sudo -s //permet de passer en mode superutilisateur

Gérer les paquets :



    apt search nom-du-paquet //permet de trouver un fichier

    apt --fix-broken install nom-du-paquet //permet de corriger des problèmes

    apt search nom-du-logiciel | grep nom (ex:PHP) //permet de recevoir et filtrer les informations reçues

    dpkg -i nom-du-paquet //permet d'installer un fichier

    touch nom-du-fichier //permet de créer un fichier

Modifier l'emplacement d'un fichier :


    mv nom-du-fichier nouveaux-nom //permet de renommer un fichier

    mv nom-du-fichier /chemin/dossier //déplacer un fichier

    mv nom-du-fichier /chemin/dossier/noveau nom-du-fichier //permet de déplacer et renommer un fichier


    nano-nom-du-fichier //permet de modifier un fichier

    rm -rf nom-du-dossier/fichier //permet de supprimer un dossier/fichier

Utilisation de git :


    git config (et git config --list) //sert à configurer git

    git clon git://lien (+ /chemin/dossier/) //clone un dépôt distant, inexistant en local(+ clone dans un répertoire)

    git status //permet de vérifier

    git .gitinore .nom-du-fichier //permet d'ignorer un fichier

    git commit nom-du-fichier //visualise si le fichier est modifié

    git diff //permet de voir ce qui a été par rapport à la dernière validation

    git diff -- cached //permet de comparer un commit avec son précédent

Créer un dépôt :


    git init //créer un dépôt local

    git add /chemin/dossier/nom-du-fichier (ou nom-du-fichier) //indexe un fichier avec son chemin (ou directement avec son nom)

    git commit -m "Nom du commit" //donne un nom à un commit, après validation (avec git add)

    git remote add origin lien-github //créer un dépôt distant

    git push origin master (nom de la branche principale)

    git push checkout nom-d'une-branche //bascule vers nom-d'une-branche

    git branch -d nom-d'une-branche //supprime la branche

    git pull origin master //récupère les fichiers d'un dépôt distant

Créer un fichier Vhost :

Se mettre en mode root. 

    cd /etc/apache2/sites-available/

    nano nom-du-fichier (nsuite cp nom-du-fichier nouveau-nom) //créer un fichier après l'avoir édité (copie/colle un fichier et le renome)

    a2ensite nom-du-fichier (ou a2dissite) //activer le fichier (ou désactiver, sert aussi aux mods)

    systemctl restart apache2 //recharge le fichier de configuration après l'avoir activé

Configurer un fichier Vhost :


    cd /etc/

    nano hosts

Ecrire 127.0.0.1 nom-du-fichier après la ligne 127.0.1.1


    systemctl restart apache2

Rendre inacessiblee certains fichiers :


    cd /etc/php/7.0/apache2

    nano php.ini
